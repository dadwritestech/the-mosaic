# Convergence SP4 — Seam-Based Map Generation Design

**Status:** Design draft 2026-06-11. The "maps first" track chosen over SP3. Produces the walkable rift-zone maps the whole reframe needs. Renders through the SP1 `MapV2` pipeline; consumed later by SP3 (gameplay wiring).

## Goal

A **structure-first procedural generator** that turns a rift (its two biomes + a seed + a few knobs) into a `MapV2` map with a visible **biome fault-line (seam)** — a guaranteed-walkable, playable map — plus a **lightweight browser editor** for hand touch-ups so each zone becomes truly authentic, not obviously procedural. One generated + tuned map per rift zone (× its state variants), for the 7 rifts.

## Decisions locked (this session)
- **Aesthetic:** seams / fault-lines (sharp boundary where world A's tiles butt against world B's), not smooth gradients.
- **Approach:** structure-first rules — lay the skeleton deterministically, then fill.
- **Authoring:** seed + parameter knobs to get ~90%, then an editor for targeted touch-ups.
- **Render target:** `MapV2` (SP1) — 3 tile-ID layers + autotiles + passages + warps + triggers + encounters + spawn.

## Art constraint (flagged, not blocking)
Outdoor art is essentially `Outside.png` + the autotiles (grass/Light grass/Flowers1-2/Sand/Sand shore/Sea*/Gravel/Brick path/Dirt/Fountain). This covers **plains, forest, beach, gardens, city** well. There is **no good outdoor lava or open-snow** tile, so **volcano (alola-islands) and tundra (sinnoh-tundra)** palettes will be approximated (red-cave/gravel for volcano; snow-cave tiles repurposed for tundra) until better tiles are sourced. Tile IDs are **harvested from the already-converted Essentials maps** (we know which IDs render as grass/tree/path/water/sand from SP1's conversions) so the generator reuses known-good IDs.

## Generator pipeline (structure-first) — SP4a

Input: `{ rift: RiftDef, seed: number, knobs: GenKnobs }`. Output: `MapV2`.

1. **Dimensions** from knobs (e.g. 28–40 wide × 24–36 tall).
2. **Seam** — a jagged fault-line (random walk seeded by `seed`, jaggedness from a knob) splitting the map into **side A** (rift `biomeA`) and **side B** (`biomeB`). Vertical or diagonal per knob.
3. **Path** — a guaranteed walkable corridor from the **entry edge** to the **exit edge**, crossing the seam once, plus a spur to the **Warden spot**. Drawn as path tiles; these cells are forced walkable.
4. **Zones** — grass/encounter patches on each side (drive wild encounters), water/feature zones, keeping the path clear.
5. **Warps & triggers** — entry/exit warps wired to the adjacent rift zones (the chain), the **Warden trigger** at its spot, optional center/shop near the entry.
6. **Fill** — each side's ground laid with its biome palette's autotile; scatter biome **features** (trees/rocks/flowers/water) by density knobs, never on path/warp cells; mark `passages` (features block, ground walks).
7. **Encounters grid** — cells in grass zones flagged so the server rolls `zoneEncounters(state, rift)` there.
8. **Emit `MapV2`** — exactly the SP1 format; the existing renderer draws it unchanged.

**Biome palette** = `{ groundAutotile, featureTiles[], accentTiles[], pathTile }` per `Biome`, in a `src/.../biome-palettes.ts` table (architect-authored from harvested IDs). Plains/forest/beach/gardens/city authentic; volcano/tundra approximated (flagged above).

**Determinism:** same `(rift, seed, knobs)` ⇒ identical map (seeded RNG, like `src/ai/rng.ts`), so a locked seed reproduces a chosen base.

## State variants
- **`unsealed`** (default) and **`attuned`** ⇒ the **seam map** (attuned's extra danger lives in `zoneEncounters`, already done in SP2; no separate map needed — optionally a knob adds hazards).
- **`sealed → biomeX`** ⇒ a **single-biome** map: same structure/seed but the whole map uses `biomeX`'s palette (no seam). Generated by running the pipeline with one palette for both sides.

So each rift needs at most: 1 seam map (unsealed/attuned) + up to 2 pure maps (sealed-A, sealed-B). All from the same generator, different palette args.

## Touch-up editor — SP4b (lightweight)
A browser tool (own page/route) that: loads a generated `MapV2`, lets you **paint tiles** (pick from the zone's palette), **place/move features, warps, the Warden, and the spawn**, toggle **passable**, and **export** the tweaked `MapV2` JSON back to `web/public/maps/<zone>.json`. Reuses the SP1 renderer for the canvas. Not a full RPG-Maker — just paint + place + export. Built after SP4a so there's something to edit.

## Region integration — SP4c
Generate + tune the 7 rift zones' maps, wire them into the region graph as walkable locations carrying their `riftId`, with entry/exit warps forming the chain (hub → rift 1 → … → rift 7 → Core). This is where the rifts become reachable (and unblocks SP3). Overlaps SP3's region rewire — coordinate so it's done once.

## Decomposition (SP4 is big — sub-pieces, each its own plan)
- **SP4a — Generator core** (structure-first pipeline + biome palettes + seeded RNG → `MapV2`). The foundation; testable headless (assert: walkable path exists entry→exit, seam present, warps/warden placed, encounters grid non-empty). ← build first.
- **SP4b — Touch-up editor** (browser paint/place/export tool).
- **SP4c — Region integration** (generate/tune the 7 zones, wire the chain; coordinate with SP3's region rewire).

## What stays untouched / reused
- SP1 `MapV2` + renderer + autotiles (consumed as-is). SP2 rift data/state (the generator reads `RiftDef.biomeA/biomeB`; the server picks the map variant by `riftStatus`). The Essentials importer demotes fully to the **tileset-harvester** that already gave us `Outside.png` + autotiles in `web/public/2d/tilesets/`.
- `src/` battle/catch/leveling/AI engine — untouched.

## Testing
- SP4a generator: headless unit tests on the emitted `MapV2` — a walkable path connects entry and exit (flood-fill over `passages`), the seam divides the two biome palettes, warps + Warden trigger + spawn are placed and on walkable cells, the encounters grid is non-empty, and generation is deterministic for a fixed `(seed, knobs)`.
- Visual verification on screen via the SP1 renderer (the seam reads as a fault-line; palettes look distinct).

## Out of scope (here)
SP3's gameplay wiring (Warden battles, the Seal/Attune choice UI, encounters actually rolling `zoneEncounters` in the server, the Core finale, badge removal). Sourcing better volcano/tundra tiles. Animated/seasonal map effects.
